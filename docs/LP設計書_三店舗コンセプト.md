# 三店舗一体型 LP 設計書

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| 目的 | 昼・夕方・夜の3コンセプト飲食店を **1枚のランディングページ** で訴求し、来店・予約・に導く |
| 前提 | **スマートフォンファースト**（幅 375px を基準設計、タブレット・PC は拡張） |
| 構成 | 縦スクロールのシングルページ（必要に応じてアンカーリンクで区切り） |

---

## 2. コンセプト別定義

### 2.1 昼（ランチ／デイタイム/ナイトタイム）

| 項目 | 仕様 |
|------|------|
| 世界観 | **木目テクスチャ** を基調とした **暖色系**（アンバー、テラコッタ、深いベージュ） |
| 料理イメージ | イタリアン系の肉料理：**サルシッチャ**、**ポルケッタ** など炭火・ローストの香ばしさを連想させるコピー・写真 |
| 訴求キーワード例 | 「昼は木の温もりと、肉の旨み」「ランチタイムのカジュアルイタリアン」 |
| 必須コンテンツ | 営業時間（昼帯）、代表メニュー名、価格帯の目安、席数のざっくり表記 |
| 店の情報 | https://tabelog.com/kanagawa/A1401/A140204/14097723/dtlphotolst/smp2/ |


### 2.2 夕方（スタンディングバー）

| 世界観 | **白木目**（ホワイトオーク／ブリーチウッド系）で照明破断承継の **スタンディングバー** |
| 訴求キーワード例 | 「仕事帰りの一杯」「立ち飲みで気軽に」 |
| 必須コンテンツ | ドリンクの強み（シャンパン・ビール・カクテル等）、立ち席の雰囲気写真、夕方〜夜中の時間帯 |

### 2.3 夜（カラオケバー）

| 世界観 | **定員おおよそ7名** の **コアなカラオケバー**（プライベート感・選曲の自由度・「知る人ぞ知る」ニュアンス） |
| 訴求キーワード例 | 「少人数で貸し切り感」「深夜の秘密基地」 |
| 必須コンテンツ | 最大収容人数（7名前後と明記）、予約の必要性、利用時間帯、音響・機材の簡単な説明 |

## 3. 情報設計（IA）

### 3.1 ページ内セクション順（推奨）

1. **ヒーロー** — 3店舗を一言で束ねるキャッチ（例：「一日を、三つの顔で」）
2. **昼** — ビジュアル＋メニュー＋時間
3. **夕方** — ビジュアル＋スタンディングの説明＋時間
4. **夜** — ビジュアル＋7名定員・カラオケの説明＋時間
5. **共通** — アクセス、SNS、電話、予約フォーム or 外部予約リンク、営業カレンダー（任意）

### 3.2 ナビゲーション

- 固定 **下部タブ** または **ハンバーガーメニュー** で「昼／夕／夜／アクセス」へジャンプ（スマホ片手操作を想定）
- 各セクション先頭に **視覚的に区切り**（背景色・区切り線・アイコン）を入れ、スクロール中に「今どの店の話か」が分かるようにする

## 4. ビジュアル／UI 設計（スマホファースト）

### 4.1 ブレークポイント方針

| 幅 | 方針 |
|----|------|
| ~480px | メインターゲット。1カラム、タップ領域 最小 44×44px |
| 481px~ | 余白・最大幅コンテナの拡張、画像の2カラム化（任意） |

### 4.2 セクション別カラーパレット（案）

| セクション | 背景・質感 | アクセント | 文字色 |
|------------|------------|------------|--------|
| 昼 | ダーク〜ミディアム木目、暖色オーバーレイ | オレンジ／テラコッタ | オフホワイト or 濃いブラウン（コントラスト確保） |
| 夕方 | **白木目**、明るいグレー白 | ネイビー or ソフトゴールド | ダークグレー |
| 夜 | 深いネイビー／チャコール、控えめなグレイン | ネオンに近い1色のみ（紫・赤の1点） | 白〜薄グレー |

※ WCAG 2.1 **コントラスト比**（本文 4.5:1 以上）を満たすこと。

### 4.3 タイポグラフィ

- 日本語：**ゴシック系**をベース（可読性）。見出しのみ **明朝 or ディスプレイ系** で世界観を出す選択可
- 欧文メニュー名（Salsiccia, Porchetta 等）は **イタリック or スモールキャップ** で統一
- 基準 **16px 以上**（本文）、行間 1.5〜1.7

### 4.4 画像・素材

- 昼：炭火・肉断面・木のテーブル等の **暖色に調整した写真**
- 夕方：**白木カウンター**、グラス、明るい店内
- 夜：**暗めのボケ**、マイク・モニターは主張しすぎず「大人の隠れ家」感

## 5. コンテンツ要件チェックリスト

- [ ] 各ゾーンに **営業時間**（重複があれば表で一本化してもよい）
- [ ] 昼：**サルシッチャ／ポルケッタ** をメニュー例として明示（写真 or テキスト）
- [ ] 夕方：**スタンディング**であることを文面で明確化
- [ ] 夜：**定員約7名** を数値で明示（「少人数のみ」等の曖昧表現だけにしない）
- [ ] 住所・最寄駅・地図リンク（Google Maps 等）
- [ ] 予約方法（電話番号、LINE、公式サイトフォーム等）

## 6. 技術・非機能要件（実装時の指針）

| 区分 | 内容 |
|------|------|
| パフォーマンス | 画像は WebP／適切な解像度、ヒーローは **LCP** を意識して優先読み込み |
| タッチ | 誤タップ防止の余白、固定フッターナビと本文の重なり回避（`padding-bottom` 確保） |
| SEO | `title` / `meta description` に3コンセプトを自然に含める、`h1` はページに1つ |
| アクセシビリティ | コントラスト、代替テキスト、フォーカス可視化 |

- 3店舗は **同一法人** で、店な別々の場所にある
- 予約は **店舗別**
